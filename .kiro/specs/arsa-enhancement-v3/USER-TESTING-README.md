## 🎯 Testing Objectives

### Primary Goals

1. **Comprehension (90% target)**
   - Users understand all dashboard elements
   - No confusion about terminology
   - Clear understanding of actions

2. **Zero Technical Jargon (100% target)**
   - No "SEO", "metadata", "slug", etc.
   - All terms in plain Indonesian
   - Friendly, accessible language

3. **Task Completion (80% target)**
   - Users can find missing information
   - Users can navigate to edit profile
   - Users can view public profile
   - Users understand admin flags

4. **Motivation (80% target)**
   - Users feel encouraged to complete profile
   - Tone is friendly and respectful
   - Users feel valued

## 📋 Testing Process

### Phase 1: Preparation (Before Testing)
1. Review all testing documents
2. Recruit participants (5-10 UMKM owners)
3. Prepare test accounts (30%, 60%, 90% completion)
4. Set up recording equipment
5. Schedule sessions (30-45 min each)

### Phase 2: Execution (During Testing)
1. **Introduction** (5 min) - Explain purpose, get consent
2. **First Impressions** (5 min) - Show dashboard, observe reactions
3. **Comprehension** (15 min) - Test understanding of each section
4. **Task Completion** (10 min) - Give specific tasks to complete
5. **Language Check** (5 min) - Verify no technical jargon
6. **Feedback** (5 min) - Collect suggestions

### Phase 3: Analysis (After Testing)
1. Review all recordings
2. Compile notes and findings
3. Calculate metrics (completion rate, comprehension, etc.)
4. Identify patterns and themes
5. Prioritize issues by severity
6. Create action plan

### Phase 4: Implementation (Fix Issues)
1. Fix critical issues immediately
2. Schedule high-priority fixes
3. Document medium/low priority for backlog
4. Re-test with 2-3 participants
5. Get final approval

## 🚨 Critical Issues to Watch For

### Red Flags (Must Fix)
- ❌ User doesn't understand main purpose
- ❌ Technical jargon causes confusion
- ❌ Cannot complete basic tasks
- ❌ Privacy concerns raised
- ❌ Feels intimidated or overwhelmed

### Yellow Flags (Should Fix)
- ⚠️ Hesitation on specific terms
- ⚠️ Multiple attempts to find information
- ⚠️ Asks for clarification repeatedly
- ⚠️ Expresses frustration
- ⚠️ Suggests simpler alternatives

### Green Flags (Good Signs)
- ✅ Immediate understanding
- ✅ Positive emotional response
- ✅ Completes tasks confidently
- ✅ Feels motivated to complete profile
- ✅ Would recommend to others

## 🎓 Testing Best Practices

### Do's ✅
- **Do** make participants comfortable
- **Do** encourage thinking aloud
- **Do** ask open-ended questions
- **Do** observe body language
- **Do** record everything (with permission)
- **Do** thank participants sincerely
- **Do** follow up on interesting comments
- **Do** stay neutral (don't defend design)

### Don'ts ❌
- **Don't** lead participants to answers
- **Don't** explain how things work (unless asked)
- **Don't** interrupt participants
- **Don't** defend design choices
- **Don't** rush through sessions
- **Don't** skip documentation
- **Don't** ignore negative feedback
- **Don't** test without consent

## 📞 Support & Questions

### Need Help?

**Before Testing:**
- Review all three documents thoroughly
- Practice with a colleague first
- Prepare test accounts in advance

**During Testing:**
- Follow the checklist
- Take detailed notes

**After Testing:**

### Common Questions

**Q: How many participants do we need?**  
A: Minimum 5, ideal 8-10. More diversity = better insights.

**Q: How long should each session be?**  
A: 30-45 minutes. Don't rush, but respect their time.

**Q: What if a participant is confused?**  
A: That's valuable data! Note what confused them and why.

**Q: Should we fix issues between sessions?**  
A: Only if critical. Better to test consistently, then fix all at once.

**Q: What if we don't meet success targets?**  
A: Identify issues, prioritize fixes, implement, and re-test.

**Q: Can we test remotely?**  
A: Yes! Use video calls with screen sharing. Get recording permission.
