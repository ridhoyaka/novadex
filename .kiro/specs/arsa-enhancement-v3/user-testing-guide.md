# ARSA Enhancement v3.0 - User Testing Guide
## Dashboard Copy Testing with Real UMKM Users

**Purpose:** Validate that the UMKM dashboard copy is understandable, free of technical jargon, and motivating for real UMKM business owners.

**Target Participants:** 5-10 real UMKM business owners with varying levels of digital literacy

**Duration:** 30-45 minutes per session

**Date Created:** 2026-02-07

---

## Testing Objectives

1. **Comprehension:** Verify users understand all dashboard elements without confusion
2. **Clarity:** Ensure copy is free of technical jargon (SEO, metadata, slug, etc.)
3. **Motivation:** Confirm messaging encourages profile completion
4. **Usability:** Validate that users can navigate and take actions easily
5. **Trust:** Ensure privacy notices and help text build confidence

---

## Pre-Testing Preparation

### 1. Participant Selection Criteria

**Ideal Participants:**
- Active UMKM business owners in Salatiga
- Mix of age groups (20s-60s)
- Varying digital literacy levels:
  - 2-3 beginners (rarely use computers/apps)
  - 3-4 intermediate (comfortable with WhatsApp, social media)
  - 1-2 advanced (use multiple business apps)
- Different business categories (food, crafts, services, retail)
- Mix of new and existing ARSA users

**Recruitment:**
- Contact through existing UMKM network
- Offer small incentive (Rp 50,000 voucher or product)
- Schedule at convenient times (avoid business hours)
- Option for in-person or remote testing

### 2. Testing Environment Setup

**For In-Person Testing:**
- Quiet room with computer/laptop
- Screen recording software enabled
- Note-taking materials ready
- Test account with sample data prepared

**For Remote Testing:**
- Video call platform (Zoom, Google Meet)
- Screen sharing enabled
- Recording permission obtained
- Test credentials sent in advance

### 3. Test Account Preparation

Create test accounts with different completion levels:
- **Account A:** 30% complete (minimal data)
- **Account B:** 60% complete (good progress)
- **Account C:** 90% complete (nearly optimal)

## Testing Protocol

### Phase 1: Introduction (5 minutes)

**Script:**
> "Terima kasih sudah bersedia membantu kami. Hari ini kami ingin mendengar pendapat Anda tentang tampilan dashboard ARSA yang baru. Tidak ada jawaban benar atau salah - kami ingin tahu pengalaman Anda yang sebenarnya.
>
> Anda boleh berbicara dengan bebas tentang apa yang Anda lihat, apa yang Anda pahami, dan apa yang membingungkan. Jika ada yang tidak jelas, itu bukan kesalahan Anda - itu berarti kami perlu memperbaiki desain kami.
>
> Saya akan merekam layar dan percakapan kita untuk keperluan analisis. Apakah Anda setuju?"

**Consent:**
- [ ] Verbal consent obtained
- [ ] Recording started
- [ ] Participant comfortable

### Phase 2: First Impressions (5 minutes)

**Task:** Show dashboard without explanation

**Questions:**
1. "Apa kesan pertama Anda saat melihat halaman ini?"
2. "Apa yang paling menarik perhatian Anda?"
3. "Apakah ada yang membingungkan atau tidak jelas?"
4. "Menurut Anda, halaman ini untuk apa?"

**Observe:**
- [ ] Eye movement patterns
- [ ] Facial expressions (confusion, understanding)
- [ ] Time to understand main purpose
- [ ] Spontaneous comments

### Phase 3: Comprehension Testing (15 minutes)

#### 3.1 Profile Completion Section

**Point to "Kelengkapan Profil Anda" section**

**Questions:**
1. "Apa yang dimaksud dengan 'Kelengkapan Profil'?"
2. "Mengapa kelengkapan profil penting menurut Anda?"
3. "Apa arti angka persentase ini (misal: 60%)?"
4. "Apa perbedaan antara 'Profil Dasar', 'Profil Lengkap', dan 'Profil Optimal'?"
5. "Apakah Anda paham apa yang harus dilakukan untuk meningkatkan kelengkapan?"

**Check Understanding:**
- [ ] Understands completion percentage
- [ ] Knows what "Profil Optimal" means
- [ ] Can identify missing fields
- [ ] Motivated to complete profile
- [ ] No confusion about technical terms

#### 3.2 Missing Fields Checklist

**Point to checklist of missing items**

**Questions:**
1. "Apa yang ditunjukkan oleh daftar ini?"
2. "Apakah Anda paham apa yang dimaksud dengan setiap item?"
   - "Deskripsi terlalu singkat (minimal 50 karakter)" - jelas?
   - "Belum ada foto usaha (minimal 1 foto dianjurkan)" - jelas?
   - "Lokasi usaha belum ditambahkan (opsional)" - jelas?
3. "Apa yang akan Anda lakukan setelah melihat daftar ini?"

**Check Understanding:**
- [ ] Understands each missing field
- [ ] Knows "opsional" means optional
- [ ] Understands character count (50 karakter)
- [ ] Motivated by friendly tone
- [ ] Clear on next action

#### 3.3 Status Indicators

**Point to status badges and messages**

**Questions:**
1. "Apa arti 'Profil Aktif' vs 'Profil Tidak Aktif'?"
2. "Apa yang terjadi jika profil tidak aktif?"
3. "Bagaimana cara mengubah status profil?"
4. "Apakah Anda paham perbedaan antara profil yang terlihat publik vs tersembunyi?"

**Check Understanding:**
- [ ] Understands publish/unpublish concept
- [ ] Knows how to toggle status
- [ ] Comfortable with control over visibility
- [ ] No privacy concerns

#### 3.4 Help Text & Tooltips

**Hover over help icons (ℹ️)**

**Questions:**
1. "Apakah Anda melihat ikon bantuan ini?"
2. "Apa yang Anda harapkan terjadi jika mengklik/hover?"
3. "Apakah teks bantuan yang muncul membantu?"
4. "Apakah bahasanya mudah dipahami?"

**Check Understanding:**
- [ ] Notices help icons
- [ ] Understands how to access help
- [ ] Help text is clear and useful
- [ ] Language is friendly and non-technical

#### 3.5 Quick Actions Cards

**Point to three action cards**

**Questions:**
1. "Apa fungsi dari tiga kotak ini?"
2. "Apa yang akan terjadi jika Anda klik 'Perbarui Profil'?"
3. "Apa yang akan terjadi jika Anda klik 'Aktifkan/Sembunyikan Profil'?"
4. "Apa yang akan terjadi jika Anda klik 'Buka Halaman'?"
5. "Apakah label-label ini jelas?"

**Check Understanding:**
- [ ] Understands each action
- [ ] Knows which action to take first
- [ ] Comfortable with terminology
- [ ] Icons are helpful

### Phase 4: Task Completion (10 minutes)

**Give specific tasks:**

#### Task 1: Find Missing Information
> "Bayangkan profil Anda baru 50% lengkap. Bagaimana Anda tahu apa yang masih kurang?"

**Success Criteria:**
- [ ] Finds completion section
- [ ] Reads missing fields list
- [ ] Understands what to do next

#### Task 2: Edit Profile
> "Anda ingin menambahkan foto usaha. Bagaimana caranya?"

**Success Criteria:**
- [ ] Clicks "Lengkapi Sekarang" or "Perbarui Profil"
- [ ] Confident in navigation
- [ ] No hesitation

#### Task 3: View Public Profile
> "Anda ingin melihat bagaimana pelanggan melihat profil Anda. Bagaimana caranya?"

**Success Criteria:**
- [ ] Finds "Halaman Publik" card
- [ ] Clicks "Buka Halaman"
- [ ] Understands it opens in new tab

#### Task 4: Understand Flags (if applicable)
> "Admin memberikan catatan tentang profil Anda. Di mana Anda bisa melihatnya?"

**Success Criteria:**
- [ ] Finds flag notification
- [ ] Understands flag message
- [ ] Knows how to respond

### Phase 5: Language & Tone Assessment (5 minutes)

**Questions:**
1. "Apakah ada kata atau istilah yang tidak Anda pahami?"
2. "Apakah bahasa yang digunakan terasa ramah atau terlalu formal?"
3. "Apakah ada yang terasa seperti 'bahasa komputer' atau terlalu teknis?"
4. "Apakah pesan-pesan di dashboard memotivasi Anda untuk melengkapi profil?"
5. "Apakah Anda merasa dihargai sebagai pemilik usaha?"

**Check for Technical Jargon:**
- [ ] No mention of "SEO"
- [ ] No mention of "metadata"
- [ ] No mention of "slug"
- [ ] No mention of "optimization"
- [ ] All terms are plain Indonesian

**Check Tone:**
- [ ] Friendly and encouraging
- [ ] Not condescending
- [ ] Motivating, not demanding
- [ ] Respectful of user's time

### Phase 6: Suggestions & Feedback (5 minutes)

**Open-ended questions:**
1. "Apa yang paling Anda sukai dari dashboard ini?"
2. "Apa yang paling membingungkan atau perlu diperbaiki?"
3. "Apakah ada informasi yang Anda harapkan ada tapi tidak ada?"
4. "Apakah ada informasi yang menurut Anda tidak perlu?"
5. "Jika Anda bisa mengubah satu hal, apa itu?"

**Document:**
- [ ] Positive feedback
- [ ] Pain points
- [ ] Suggestions for improvement
- [ ] Unexpected insights

## Analysis Framework

### Quantitative Metrics

**Success Rate Targets:**
- Task completion: ≥ 80% success rate
- Comprehension: ≥ 90% "Clear" ratings
- No technical jargon: 100% (zero tolerance)
- Overall satisfaction: ≥ 80% "Yes" or "Maybe"

**Calculate:**
- Average completion time per task
- Number of errors per participant
- Number of help requests
- Confusion points (where users hesitate)

### Qualitative Analysis

**Themes to identify:**
1. **Confusion Points:** Where do users get stuck?
2. **Delight Moments:** What do users love?
3. **Language Issues:** Which terms are unclear?
4. **Missing Information:** What do users expect but don't find?
5. **Unexpected Behaviors:** What surprises users?

**Severity Ratings:**
- **Critical:** Blocks task completion, must fix
- **High:** Causes confusion, should fix
- **Medium:** Minor inconvenience, nice to fix
- **Low:** Cosmetic, optional fix

## Post-Testing Actions

### 1. Consolidate Findings
- [ ] Review all recordings
- [ ] Compile notes from all sessions
- [ ] Identify patterns and themes
- [ ] Calculate metrics

### 2. Prioritize Issues
- [ ] Categorize by severity
- [ ] Estimate effort to fix
- [ ] Determine impact on users
- [ ] Create action plan

### 3. Implement Changes
- [ ] Fix critical issues immediately
- [ ] Schedule high-priority fixes
- [ ] Document medium/low priority for backlog
- [ ] Update copy based on feedback

### 4. Validate Changes
- [ ] Review changes with 2-3 participants
- [ ] Confirm issues are resolved
- [ ] No new issues introduced
- [ ] Get final approval

### 5. Document Learnings
- [ ] Update design guidelines
- [ ] Create copy writing standards
- [ ] Share insights with team
- [ ] Plan for continuous testing

## Success Criteria

**Dashboard copy is ready for production when:**

✅ **Comprehension:**
- 90%+ participants understand all sections
- Zero technical jargon confusion
- All terms are plain Indonesian

✅ **Usability:**
- 80%+ task completion rate
- No critical blockers
- Users confident in navigation

✅ **Motivation:**
- 80%+ participants feel motivated
- Tone is friendly and encouraging
- Users feel respected and valued

✅ **Trust:**
- Privacy notices are clear
- Users comfortable with data sharing
- No security concerns raised

✅ **Overall:**
- 80%+ would recommend to other UMKM
- No major revision needed
- Ready for production deployment
